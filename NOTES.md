# DashBasket Round 2 Notes

## Main idea
In round 1, we made DashBasket intentionally frustrating.

In round 2, we redesigned the same app to make it clear, usable, and trustworthy.

The main focus was not just making it look better.
The main focus was fixing the actual user problems.

## What we fixed
- search is now direct and unbiased
- filters and sort behave properly
- exact results are shown clearly
- product page opens directly without forced ad
- pricing is shown in one clear way
- sizes are labeled properly with real quantities
- cart only shows user-added items
- totals are stable and transparent
- payment selection stays as selected
- checkout has one clear final action
- order confirmation is simple and final

## Screen-wise before and after

### Home
Before:
- too many distractions
- search was biased
- offers came before clarity

After:
- clear hero section
- strong search at the top
- simple category entry points
- easier start for the user

### Browse
Before:
- sponsored and wrong items came first
- filters looked normal but behaved badly
- result count was misleading

After:
- clean filter row
- relevant results only
- correct sorting
- honest result count

### Product
Before:
- forced ad before opening
- conflicting prices
- unclear sizes
- too many distracting badges

After:
- direct product access
- one main price and one MRP
- actual size labels like 500ml, 1L, 2L
- useful details only

### Cart
Before:
- auto-added products
- hidden charges and changing totals
- savings shown more loudly than total

After:
- only user-selected items
- simple bill summary
- stable total
- clear next step

### Checkout
Before:
- payment changed unexpectedly
- CTA changed
- extra verification and confusion
- contradictory order state

After:
- stable address and payment section
- one clear place order button
- no unnecessary interruption
- final success message is clear

## What judges should notice
- lower cognitive load
- more user control
- better trust
- better clarity
- more predictable flow
- smoother checkout experience

## 3 minute presentation flow
1. Briefly explain that round 1 was intentionally designed to frustrate users.
2. Mention the biggest issues: biased search, confusing product page, unstable cart, and stressful checkout.
3. Show how each screen was redesigned for clarity and trust.
4. End by saying the redesign improves usability, accessibility, and confidence.

## Q&A support points
If judges ask why this redesign is better:
- users can now predict what will happen next
- product information is now honest and readable
- totals do not change unexpectedly
- checkout feels safe and consistent

If judges ask about accessibility:
- clearer hierarchy
- better spacing
- fewer distractions
- readable text and simpler interaction flow

## One line summary
Round 1 was about making DashBasket stressful and misleading. Round 2 fixes that by making the same app transparent, simple, and trustworthy.
